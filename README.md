# CMS System - Система управления контентом

Полнофункциональная система управления контентом (CMS), разработанная на PHP с использованием MySQL. Предназначена для управления веб-сайтами малого и среднего размера.

## Требования

- PHP 7.4 или выше
- MySQL 5.7 или выше
- Apache с включённым mod_rewrite или Nginx
- 50MB свободного место для загрузок

## Установка

### 1. Подготовка базы данных

```bash
# Создайте базу данных MySQL
mysql -u root -p
CREATE DATABASE cms_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
EXIT;

# Импортируйте схему
mysql -u root -p cms_db < database/schema.sql
```

### 2. Конфигурация

Отредактируйте файл `config/database.php`:

```php
return [
    'host' => 'localhost',
    'database' => 'cms_db',
    'username' => 'root',
    'password' => '', // Ваш пароль MySQL
];
```

### 3. Настройка веб-сервера

#### Для Apache:
- Включите mod_rewrite: `a2enmod rewrite`
- Убедитесь, что `.htaccess` находится в папке `public/`

#### Для Nginx:
```nginx
location / {
    if (!-e $request_filename) {
        rewrite ^(.*)$ /index.php last;
    }
}

location ~ \.php$ {
    fastcgi_pass unix:/var/run/php-fpm.sock;
    fastcgi_index index.php;
    include fastcgi_params;
    fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
}
```

### 4. Права доступа

```bash
chmod 755 public/
chmod 755 storage/
chmod 755 storage/uploads/
chmod 755 storage/logs/
chmod 755 storage/backups/
chmod 755 database/
```

### 5. Первый пользователь

После установки создайте администратора через SQL:

```sql
INSERT INTO users (name, email, password, role, status, created_at) VALUES (
    'Admin',
    'admin@example.com',
    '$2y$12$...',  -- BCRYPT хеш пароля 'password'
    'admin',
    'active',
    NOW()
);
```

Для хеширования пароля используйте:
```php
echo password_hash('password', PASSWORD_BCRYPT, ['cost' => 12]);
```

## Использование

### Вход в систему

1. Перейдите на `http://yourdomain.com/login`
2. Введите email и пароль администратора
3. Нажмите "Вход"

### Основные функции

#### Управление постами
- Создание новых статей/новостей
- Сохранение в статусе черновик
- Опубликование постов
- Редактирование и удаление

#### Управление категориями
- Создание иерархических категорий
- Редактирование описаний
- Удаление категорий (если нет постов)

#### Управление медиа
- Загрузка изображений, видео, документов
- Организация медиабиблиотеки
- Удаление файлов

#### Управление комментариями
- Модерация комментариев
- Одобрение/отклонение
- Удаление спама

#### Управление пользователями
- Создание аккаунтов
- Назначение ролей (Автор, Редактор, Администратор)
- Изменение статуса пользователя

## Роли и права

### Администратор (admin)
- Полный доступ ко всем функциям
- Управление пользователями
- Настройка системы
- Резервное копирование

### Редактор (editor)
- Управление всеми постами
- Управление категориями
- Управление медиа
- Модерация комментариев

### Автор (author)
- Создание собственных черновиков
- Просмотр статистики
- Загрузка медиа
- Редактирование своих постов

## Безопасность

### Защита от атак

- **SQL-инъекции**: Использование параметризованных запросов (PDO)
- **XSS-атаки**: Автоматическое экранирование вывода (htmlspecialchars)
- **CSRF**: Генерация и проверка токенов для всех форм

### Пароли

- Хеширование алгоритмом BCRYPT (cost 12)
- Минимальная длина: 8 символов
- Защита от перебора

### Файлы

- Проверка MIME-типов
- Проверка расширений файлов
- Ограничение размера файла (по умолчанию 50MB)
- Изоляция в папке uploads/

## Производительность

### Оптимизация

- Кеширование настроек в памяти
- Индексы на часто используемых полях
- Пагинация результатов поиска
- Минимизация запросов к БД

### Характеристики

- Время генерации страницы: < 0.5 сек
- Поддержка: ≥ 100 одновременных пользователей
- Архитектура: модульная, расширяемая

## Логирование

Все ошибки и важные события логируются в `storage/logs/{date}.log`:

```
[14:25:30] [INFO] User logged in: admin@example.com
[14:26:15] [ERROR] Post creation failed: Invalid data
[14:27:00] [WARNING] Large file upload: user_id=5
```

## Расширение функционала

### Создание нового контроллера

```php
<?php
class MyController {
    public function myAction() {
        AuthMiddleware::checkAuth();
        // Ваш код
    }
}
```

### Добавление маршрута

В файле `public/index.php`:
```php
Router::get('/myroute', function() {
    (new MyController())->myAction();
});
```
