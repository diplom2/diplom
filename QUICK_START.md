# Быстрый старт CMS

Следуйте этому пошаговому руководству для быстрого развертывания системы.

---

## ⚡ За 5 минут до работающей CMS

### Шаг 1: Подготовка сервера (2 минуты)

#### На локальной машине:
1. Откройте терминал в папке проекта `cms/`
2. Запустите установочный скрипт:

**Windows:**
```bash
install.bat
```

**Linux/Mac:**
```bash
bash install.sh
```

#### На хостинге:
1. Загрузите все файлы на сервер через FTP
2. Создайте БД MySQL
3. Импортируйте `database/schema.sql` в БД
4. Скопируйте `config/database.php.example` → `config/database.php`
5. Отредактируйте `config/database.php` с учетными данными вашей БД
6. Задайте права на папках:
   ```bash
   chmod 755 storage
   chmod 755 public/uploads
   chmod 755 storage/logs
   chmod 755 storage/backups
   ```

---

### Шаг 2: Инициализация БД (2 минуты)

Импортируйте SQL файл в вашу БД:

```sql
-- Импортировать database/schema.sql
-- Затем импортировать database/seed_admin.sql
```

Это создаст все таблицы и администратора по умолчанию.

---

### Шаг 3: Вход в систему (1 минута)

1. Откройте браузер
2. Перейдите на `http://localhost/cms/login` (или ваш домен)
3. Введите учетные данные:
   - **Email:** admin@example.com
   - **Пароль:** admin123

**ВАЖНО:** Сразу же измените пароль администратора!

## 📋 Основные операции

### Изменить пароль администратора

1. Нажмите на аватар в правом верхнем углу
2. Выберите "Профиль"
3. Измените пароль
4. Сохраните

### Создать новый пост

1. Перейдите в "Посты"
2. Нажмите "Создать пост"
3. Заполните:
   - Название
   - Содержание
   - Категория (опционально)
   - Избранное изображение (опционально)
4. Сохраните как черновик или опубликуйте

### Создать категорию

1. Перейдите в "Категории"
2. Нажмите "Создать категорию"
3. Заполните:
   - Название
   - Описание (опционально)
   - Родительская категория (опционально)
4. Создайте

### Загрузить изображение

1. Перейдите в "Медиа"
2. Перетащите изображение в зону (или нажмите для выбора)
3. Изображение загрузится автоматически

### Создать пользователя

1. Перейдите в "Пользователи"
2. Нажмите "Создать пользователя"
3. Заполните:
   - Имя
   - Email
   - Пароль
   - Роль
4. Создайте

|------|-----------|-----------|

## 🐛 Развитие ошибок

### CMS не отображается
✅ Проверьте: `public/test.php`  
✅ Убедитесь в подключении к БД  
✅ Проверьте логи: `storage/logs/`

### Белый экран при входе
✅ Включите отладку в `config/app.php` (debug = true)  
✅ Проверьте логи ошибок

### Медиа не загружается
✅ Проверьте права на `public/uploads/`  
✅ Проверьте размер файла
✅ Проверьте поддерживаемые форматы

### Постоянная перз на логин
✅ Проверьте сессии PHP  
✅ Проверьте cookies в браузере

## 🔐 Безопасность

### Сразу при развертывании

1. ✅ **Измените пароль администратора**
2. ✅ **Удалите файл `public/test.php`** (в production)
3. ✅ **Измените `config/database.php`** (пароли БД)
4. ✅ **Установите `secure.php`**:
   ```php
   // config/app.php
   'debug' => false, // Отключите отладку
   'session_secure' => true, // Только HTTPS
   ```

### Регулярно

- 🔄 Проверяйте логи ошибок
- 🔄 Обновляйте PHP и MySQL
- 🔄 Резервное копирование БД
- 🔄 Удаляйте старые логи

## 📞 Часто задаваемые вопросы

### Q: Как создать резервную копию БД?
**A:** Перейдите в "Настройки" → "Резервное копирование" → "Создать резервную копию". Файл будет в `storage/backups/`

### Q: Где найти логи?
**A:** `storage/logs/YYYY-MM-DD.log` (каждый день новый файл)

### Q: Как запросить новую функцию?
**A:** Прочитайте `CONTRIBUTING.md`

### Q: Как создать плагин?
**A:** Поддержка плагинов запланирована на v2.0

### Q: Можно ли использовать другую БД?
**A:** Текущая версия работает только с MySQL. PostgreSQL поддержка в v2.0

### Q: Как обновить CMS?
**A:** Смотрите `CHANGELOG.md` для инструкций
