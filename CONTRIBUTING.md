# CONTRIBUTING.md

# Способ внести вклад в CMS System

Спасибо за интерес к развитию проекта! Это документ описывает как помочь с развитием.

## Процесс разработки

1. **Форк проекта** на GitHub
2. **Создайте ветку** для вашей функции (`git checkout -b feature/АМояФункция`)
3. **Сделайте коммиты** изменений (`git commit -m 'Добавить новую функцию'`)
4. **Push ветку** на GitHub
5. **Откройте Pull Request**

## Стандарты кода

### PHP
- Используйте PSR-12 для стиля кода
- Документируйте функции с PHPDoc
- Избегайте глобальных переменных
- Используйте type hints когда возможно

```php
<?php
/**
 * Получает пост по ID
 * 
 * @param int $id ID поста
 * @return array|null Данные поста или null
 */
public function getPost(int $id): ?array
{
    return $this->db->fetchOne(
        "SELECT * FROM posts WHERE id = ?",
        [$id]
    );
}
```

### HTML/CSS/JavaScript
- Используйте семантический HTML5
- Избегайте inline стилей
- Минимизируйте jQuery, используйте Vanilla JS
- Мобильный-первый подход

### Git Commits
```
[КАТЕГОРИЯ] Краткое описание изменения

Подробное описание если необходимо.

Fixes #123
```

Категории: FEATURE, BUGFIX, DOCS, REFACTOR, STYLE, TEST

## Отчеты об ошибках

При создании Issue включите:
1. Версия PHP и MySQL
2. ОС и веб-сервер
3. Шаги для воспроизведения
4. Ожидаемое поведение
5. Фактическое поведение
6. Логи из `storage/logs/`

## Функции и улучшения

За новыми функциями:
1. Создайте Issue для обсуждения
2. Ждите одобрения от мейнтейнеров
3. Начните разработку
